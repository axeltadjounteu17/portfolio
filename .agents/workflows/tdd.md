---
description: Appliquer la méthodologie Test-Driven Development (RED-GREEN-REFACTOR).
---
1. **RED** : Écrire un test unitaire pour la fonctionnalité à implémenter. Vérifier qu'il échoue.
2. **GREEN** : Écrire le code minimal nécessaire pour faire passer le test.
3. **REFACTOR** : Améliorer la structure du code sans changer son comportement. Vérifier que les tests passent toujours.
4. Documenter les tests effectués.
